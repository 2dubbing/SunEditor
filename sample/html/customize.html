Add new module object

1. It is divided into two folders dialog and submenu

    
    
  1. dialog : It is a module in the form of a dialog
        Add the {moduleName}.js file to the 'plugins/dialog/' directory
        
    2. 
    
  2. submenu : It is a module in the form of a submenu
         Add the {moduleName}.js file to the 'plugins/submenu/' directory
        
    3.

2. How to create {moduleName} .js

    
    
  1. Create a {moduleName}.js file with the same name as the module you want to add to the dialog or submenu folder
    2. 
    
  2. SUNEDITOR.plugin.{moduleName} Declare an Object and declare it within this Object
Adds an element, adds a function to a module, and registers an event
    3. 
    
  3. In common, all module objects must declare a function named add
      4. 
    
    • The add function takes the SUNEDITOR.editor Object and the element of the module button as an argument value
      • 
    
    • The argument values of the dialog module  : SUNEDITOR.editor Object
      • 
    
    • The argument values of the dialog submenu : SUNEDITOR.editor Object, element of the module button
      • 
    
    • The SUNEDITOR.editor object contains the commonly available function and variable values, context
      • 
    
    • SUNEDITOR.editor.context contains the elements, variable values, and user options that suneditor should remember
      • 
    
    • If you need to store the value in SUNEDITOR.editor.context
SUNEDITOR.editor.context.{moduleName} Declare the Object and use it
      • 

    • If you need to use the SUNEDITOR.editor Object in a function in a module
Call this function by replacing the this value of the function
with the SUNEDITOR.editor Object using the call function

      • 

    • It is recommended to keep the this value of all functions in the module except the add function as SUNEDITOR.editor

3. add function definition

    
    
  • Include {moduleName}.js file when module button is first clicked
    • 
    
  • This function is executed only once when the {moduleName}.js file is loaded.
    • 
    
  • It has a SUNEDITOR.editor Object as its first argument value
    • 
    
  • If it is a module in the submenu, it will have the second element value of the button of the additional clicked button
    • 
    
  • element creation and event registration Define the action to be inserted into the suneditor Dom
    • 
    
  • The element and variable values that need to be saved use that object by declaring SUNEDITOR.editor.context.{moduleName} Object
    •

4. When adding a dialog module

    
    
        
  1. The module in the dialog folder receives only SUNEDITOR.editor Object as argument value in add function.
    2. 
        
  2. After creating a tag element
      3. 
        
    • Insert the element created in SUNEDITOR.editor.context.{moduleName}.modal
    
        
  3. When you need to focus immediately after the Dialog window appears
      4. 
        
    • In SUNEDITOR.editor.context.{moduleName}.focusElement, insert the element that should have focus
    
        
  4. element creation, event registration, and insert the created element into context.dialog.modal using appendChaild function
    5.

5. When adding a submenu module

    
    
        
  1. Modules in the submenu folder receive the SUNEDITOR.editor Object and the module button element as argument values in the add function
    2. 
        
  2. The top-level div that contains the submenu element must contain 'layer_editor' in the class name
    3. 
        
  3. element creation, event registration, and insert the created element into targetElement.parentNode using appendChaild function
    4.

6. Added addModuleButtons option when creating Suneditor

    
    
  • Add button information to the optional addModuleButtons property

    addModuleButtons: [{
        buttonClass:'', // className to be added to the button
        title:'Test', // title property
        dataCommand:'test', // The js file name of the added module
        dataDisplay:'submenu', // The directory location under the plugins folder of the added module ('dialog', 'submenu')
        dataOption:'', // Options for whether the range of the dialog is inside the editor or for the entire screen
                       // If set to 'full', the entire screen will be used ('', 'full')
        innerHTML:'<div class="ico_test"></div>' // The source into the button's innerHTML (icon image)
     }]

    • 
    
  • If you want to add button options and adjust the position to where you want, see the createToolBar function
    •

모듈 추가

1. dialog, submenu 두개의 폴더로 구분됩니다.

    
    
  1. dialog : dialog 형태의 모듈입니다.
        plugins/dialog/ 경로 에 {moduleName}.js 파일을 추가합니다.
        
    2. 
    
  2. submenu : subMenu 형태의 모듈입니다.
         plugins/submenu/ 경로 에 {moduleName}.js 파일을 추가합니다.
        
    3.

2. {moduleName}.js 작성 방법

    
    
  1. dialog 또는 submenu 폴더에 추가하고자 하는 모듈과 같은 이름으로 {moduleName}.js 파일을 만듭니다.
    2. 
    
  2. SUNEDITOR.plugin.{moduleName} Object를 선언하고 이 Object 안에서
element 추가, 모듈의 기능 function 추가, event 등록등의 모든 동작을 정의합니다.
    3. 
    
  3. 공통적으로 모든 모듈 객체는 add 라는 이름의 함수가 꼭 선언되어있어야 합니다.
      4. 
    
    • add 함수는 SUNEDITOR.editor Object 와 모듈 버튼의 element 를 인자값으로 받습니다.
      • 
    
    • dialog 모듈의 인자값  : SUNEDITOR.editor Object
      • 
    
    • submenu 모듈의 인자값 : SUNEDITOR.editor Object, 모듈 버튼의 element
      • 
    
    • SUNEDITOR.editor 객체에는 공통적으로 사용가능한 function 및 변수값
그리고 suneditor 의 element 가 들어있는 context Object 가 있습니다.
      • 
    
    • SUNEDITOR.editor.context 에는 해당 editor 에서 필요한 값을 저장해야 하는 element 및 변수값이 들어있습니다.
      • 
    
    • 만약 SUNEDITOR.editor.context 에 값을 저장해야 한다면
    SUNEDITOR.editor.context.{moduleName} 의 Object 를 선언 후 사용하세요.
      • 

    • 만약 모듈안의 함수에서 SUNEDITOR.editor Object 를 사용해야 한다면
call 함수를 이용하여 해당 함수의 this 값을 SUNEDITOR.editor Object 로 바꿔서 호출해주세요.
* 모듈안에서 add 함수를 제외한 모든 함수들의 this 값은 SUNEDITOR.editor 로 유지하는게 좋습니다.

3. add function 정의

    
    
  • 모듈 버튼이 처음 클릭됐을때 {moduleName}.js 파일을 include 합니다.
    • 
    
  • 모듈 파일이 로드 됐을때 최초 한번만 실행되는 함수 입니다.
    • 
    
  • 이 함수는 첫번째 인자값으로 SUNEDITOR.editor Object 를 가집니다.
    • 
    
  • 만약 submenu 안의 모듈이라면 추가로 모듈 버튼의 element 를 두번째 인자값으로 가집니다.
    • 
    
  • 이 함수 안에서 element 생성 및 event 등록 에디터 Dom에 삽입 하는 동작까지 정의 합니다.
    • 
    
  • 저장해야 하는 element 및 변수값들은 SUNEDITOR.editor.context.{moduleName} Object 를 선언하여 해당 오브젝트를 사용합니다.
    •

4. dialog module 추가시

    
    
        
  1. dialog 폴더내의 모듈은 add 함수에서 SUNEDITOR.editor Object 만 을 인자값으로 받습니다.
    2. 
        
  2. tag element 생성 후
      3. 
        
    • SUNEDITOR.editor.context.{moduleName}.modal 에 생성된 엘리먼트를 넣어주세요.
    
        
  3. Dialog 창이 뜬 후 곧바로 포커싱을 줘야할때
      4. 
        
    • SUNEDITOR.editor.context.{moduleName}.focusElement 에 포커스가 가야할 엘리먼트를 넣어주세요.
    
        
  4. element 생성, event 등록 후 생성한 element 를 context.dialog.modal 에 appendChaild 함수를 이용하여 삽입 합니다.
    5.

5. submenu module 추가시

    
    
        
  1. submenu 폴더내의 모듈은 add 함수에서 SUNEDITOR.editor Object 와 모듈 버튼 element 를 인자값으로 받습니다
    2. 
        
  2. submenu element 가 들어있는 최상위 div는 layer_editor 가 클래스 이름에 포함되어 있어야 합니다.
    3. 
        
  3. element 생성, event 등록 후 생성한 element 를 targetElement.parentNode 에 appendChaild 함수를 이용하여 삽입하면 끝납니다.
    4.

6. Suneditor 생성시 addModuleButtons 옵션 추가

    
    
  • 옵션의 addModuleButtons 속성에 버튼 정보를 추가해줍니다.

    addModuleButtons: [{
        buttonClass:'', // 버튼에 추가할 클래스명
        title:'Test', // title 속성
        dataCommand:'test', // 추가한 모듈의 js 파일이름
        dataDisplay:'dialog', // 추가한 모듈의 폴더 이름 ('dialog', 'submenu')
        dataOption:'', // dialog 형태의 모듈일때 dialog의 범위가 에디터 안쪽인지 화면 전체로 설정할지 에 대한 옵션
                       // 'full'로 설정하면 화면 전체를 사용합니다 ('', 'full')
        innerHTML:'<div class="ico_test"></div>' // 버튼안에 들어갈 innerHTML 코드 (아이콘)
     }]

    • 
    
  • 버튼 옵션을 추가하고 위치를 원하는 곳으로 조정하고 싶다면 createToolBar 함수를 참고하십시오
    •
