2) 모듈 js 작성 방법

    
    
  1. dialog 또는 submenu 폴더에 추가하고자 하는 모듈과 같은 이름으로 js 파일을 만듭니다.
    2. 
    
  2. SUNEDITOR.plugin.{moduleName} 의 Object를 선언하고 이 Object 안에서
element 추가, 모듈의 기능 function 추가, event 등록등의 모든 동작을 정의합니다.
    3. 
    
  3. 공통적으로 모든 모듈 객체는 add 라는 이름의 함수가 꼭 선언되어있어야 합니다.
      4. 
    
    • add 함수는 SUNEDITOR.editor Object 와 클릭한 버튼의 element 를 인자값으로 받습니다.
      • 
    
    • dialog 모듈의 인자값  : SUNEDITOR.editor Object
      • 
    
    • submenu 모듈의 인자값 : SUNEDITOR.editor Object, 클릭한 버튼의 element
      • 
    
    • SUNEDITOR.editor 객체에는 공통적으로 사용가능한 function, 변수값, element 가 들어있는 context Object 가 들어있습니다.
      • 
    
    • SUNEDITOR.editor.context 에는 해당 editor 에서 필요한 값을 기억해야할 element 및 변수값이 들어있습니다.
      • 
    
    • 만약 SUNEDITOR.editor.context 에 값을 저장해야 한다면
    SUNEDITOR.editor.context.{moduleName} 의 오브젝트를 선언 후 사용하세요.
      • 

    • 만약 모듈안의 함수에서 SUNEDITOR.editor Object 를 사용해야 한다면
call 함수를 이용하여 해당 함수의 this 값을 SUNEDITOR.editor Object 로 바꿔서 호출해주세요.
* 모듈안에서 add 함수를 제외한 모든 함수들의 this 값은 SUNEDITOR.editor 로 바꿔서 호출하는게 좋습니다.
    




3) add function 정의

    
    
  • 모듈 js 파일 호출시 최초 한번 실행되는 함수 입니다.
    • 
    
  • 이 함수는 첫번째 인자값으로 SUNEDITOR.editor Object 를 받습니다.
    • 
    
  • 만약 submenu 안의 모듈이라면 추가로 클릭한 버튼의 element 를 두번째 인자값으로 받습니다.
    • 
    
  • 이 함수 안에서 element 생성 및 event 등록 에디터 Dom에 삽입 하는 동작까지 정의 합니다.
    • 
    
  • 만약 dialog 안의 모듈이라면 생성한 element 를 SUNEDITOR.context.dialog.modal 에 넣어주어야 합니다.
    • 
    
  • 기억해야할 element 및 변수값들은 SUNEDITOR.editor.context.{moduleName} 오브젝트를 선언하여 해당 오브젝트를 사용합니다.
    •
